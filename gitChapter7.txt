1. 스태시(stash)
  - 정의 : 완료되지 않은 작업(커밋되지 않은 변경 내용)이 남아 있을 때, 현재 작업을 임시로 저장하는 기능 
  - 현재 워킹 디렉터리 내역을 별도의 스택 영역에 잠시 보관하는 기능
  - 스태시 명령을 실행하면 현재 작업 중인 내용은 임시 저장되고, 수정 전 마지막 커밋 상태로 돌아감 
    => 이전 커밋 후 작업하지 않은 상태의 워킹 디렉토리
  - git stash
  - git stash save "WIP : 메시지~~" : 스태시가 여러 개 있을 때 구별을 위한 메시지 추가
  - [--keep-index] : 스테이지 영역의 파일들을 제외하고 스태시 생성
  - [--include-untracked] : 등록되지 않은 untracked 상태의 파일을 스태시로 생성할 경우
  - git stash list : 스태시에 저장된 스택 조회
  - git stash pop : 스택에 저장된 제일 마지막에 저장된 내용을 읽어옴
    => 스택에서 내용을 읽어 올 때 현재 브랜치의 워킹 디렉터리와 자동으로 병합
  - 스태시 복원으로 충돌 : 스태시 복원 시 충돌이 생기면 직접 문제 해결
  - 복원 도중 충돌이 생기면 스태시는 스택에 저장된 내용을 자동으로 삭제하지 않음
    => 직접 충돌 해결 후 스태시 목록을 수동을 ㅗ삭제
  - git stash branch 브랜치이름 : 새로운 브랜치를 생성한 후 스태시를 적용하는 방법
  1.1 스태시 복사
   - 스태시 스택에 저장된 항목들을 어느 브랜치에서나 복원 가능 
   - git stash apply 
   - pop 명령어 : 스택 내용을 복원한 후 스택 목록에서 자동으로 삭제 
     => 스택 내용을 워킹 디렉터리로 이동하는 것과 동일
   - apply 옵션 : 스택 목록을 읽은 후 자동으로 삭제하지 않음 
     => 스태시 내용을 워킹 디렉터리로 복사하는 것
  1.2 스태시 삭제
   - 스태시 충돌 or apply 명령어로 워킹 디렉토리를 복구할 때 스택에서 자동 삭제 x
    => git stash drop 

2. 워킹 디렉토리 청소
  - git clean : 워킹 디렉터리에 있는 추적되지 않는 파일들을 찾아서 삭제
  - [-f] : 추적하지 않는 파일들을 강제 삭제
  - [-n] : clean 명령어를 사용해 잘못 삭제하는 것을 미연에 방지하고자 파일을 가상으로 미리 처리해보고 사용자에게 확인 요청
  - [-d] : 추적되지 않는 파일만 별도로 삭제 / untracked 상태의 파일만 삭제
  - [-x] : .gitignore 파일까지 같이 삭제하고 싶을 경우

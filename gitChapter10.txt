1. 태그
 - 코드 배포를 관리할 때 사용하느 ㄴ기능
 - 깃은 정리된 커밋을 배포할 수 있도록 특수한 포인터를 제공하고, 특정 커밋을 가리키는 포인터로 버전을 관리
   => 이 포인터를 태그라고 함
   => 태그는 특정 커밋의 해시 값을 가리키는 꼬리표
 - 태그 구분
   => Annotated : 태그 이름 + 정보 포함
   => Lightweight : 태그 이름만 포함

2. Annotated 태그
 - Annotated 태그 생성 시 커밋의 해시 값뿐 아니라 생성자 정보를 같이 넣을 수 있음
 - git tag -a 버전 
   => 메시지 작성 후 저장 시 정상적으로 태그 생성
   => 태그는 현재의 마지막 커밋을 기준으로 생성 / 이 커밋은 HEAD 포인터와 일치
 2.1 간단한 메시지
  - git tag -a 1.1.0 -m [message 내용] :tag 생성 + 메시지 추가
 2.2 태그는 중복해서 생성 불가
  - 중복된 태그 이름을 사용하지 못하도록 태그 목록을 검사함
 2.3 태그 삭제
  - git tag -d 태그 이름 
 2.4 태그의 상세 정보 확인 : show 명령어
  - git show 태그 이름

3. Lightweight 태그는 
 3.1 체크섬
  - 깃의 체크섬만 가지고 있는 태그
  - 태그와 관련된 옵션이 없음
  - git tag 태그이름

4. 특정 커밋 태그는
 4.1 특정 커밋 태그
  - 현재 HEAD 포인터를 가리키는 커밋이 아닌 특정 커밋을 직접 지정해 태그 생성
  - git tag -a 태그버전 커밋ID

5. 태그를 사용한 체크아웃
  - git checkout 태그이름
  - 태그를 사용해 특정 커밋으로 체크아웃할 수도 있지만 브랜치처럼 추가로 커밋을 작성할 수는 없음
    => 태그는 브랜치와 달리 단순 포인터이기 때문
 5.1 태그 브랜치 
  - 추가 커밋 작업이 필요하다면 태그를 기반으로 새 브랜치를 생성
  - git checkout -b 브랜치이름 태그이름 
   => 태그 이름으로 새 브랜치를 생성하고, 체크아웃까지 동시에 하기

6. 태그 공유 
  - 원격 저장소에 커밋 전송 => 태그 정보는 전송에서 제외됨 => 태그는 별도 전송해야함
  - git push  태그이름 - 태그 전송하기
  - github repository에 tag 숫자 확인하면 전송 여부 확인 가능
  6.1 전체 태그 동기화
  - git push origin --tags : 로컬 저장소의 모든 태그를 한꺼번에 원격 저장소로 보내기 
  6.2 원격 저장소의 태그 수정과 삭제
  - git push --delete 서버이름 태그이름 
   => 원격 저장소로 공유된 태그 삭 시 push 명령어 사용
  - 태그는 저장소 간 자동으로 동기화 하지 않음
  - 전송된 태그 이름 변경 시 
   => 원격 저장소의 태그와 로컬 저장소의 태그 모두 삭제 => 로컬 저장소에서 다시 새로운 태그 생성
   => 원격 저장소로 생성된 태그를 재전송
  - 원격 저장소의 태그 목록을 가져올 때는 pull 사용
  - clone 시도 태그 정보도 같이 가져옴
  6.3 원격 저장소에 로컬과 다른 이름으로 태그 전송
  - git push origin 태그이름 : 원격저장소의 태그이름
   => 로컬 저장소의 태그 이름과 원격 저장소의 태그 이름 지정하기

7. 태그 생성 원리
 - 깃은 태그 생성 정보를 ./git/ref/tags 폴더에 저장
 - echo [commit 멘트] > ./git/refs/tags/[태그 이름]

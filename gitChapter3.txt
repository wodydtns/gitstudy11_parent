1. 워킹 디렉터리
  - 깃은 저장 공간을 '작업을 하는 공간(working)' - '임시로 저장하는 공간(stage)' & '실제로 저장하여 기록하는 공간(respository)'
  - 정의 : 작업을 하는 공간

2. 파일의 untracked 상태와 tracked 상태
  2.1 untracked 상태
   - 워킹 디렉토리에 새로 생성된 파일은 모두 untracked 상태
   - git에 통지하지 않은 파일은 깃에서 따로 추적하지 않는다
  2.2 tracked 상태
   - 추적되지 않은 상태의 파일을 tracked 상태로 변경
    => git add [filename]

3. 스테이지
  - tracked 파일들을 관리
  - 워킹 디렉토리와 레파지토리 사이의 임시 영역
  3.1 파일의 stage 상태와 unstage 상태
   - 스테이지 영역에 등록된 모든 파일은 untracked => tracked 상태로 변경
   - 깃이 변화 이력을 기록하려면 파일들의 최종 상태가 stage 상태여야함
   - unstage는 파일에 변화가 있다는 것을 의미
    => 스테이지 영역에 있는 파일과 워킹 디렉토리 안에 있는 파일 내용에 차이가 있을 때 unstage 사애
   - unstage 상태가 실제 파일이 없어지는 것은 아니고 파일이 수정되 임시적으로 스테이지 목록에서 제외된 것
    => git add 하면 스테이지에 다시 추가
  3.2 파일의 modified 상태와 unmodified 상태
   -modified 상태 : tracked 상태인 파일이 수정되면 스테이지가 파일 상태를 modified 상태로 변경
    -> 수정된 파일은 스테이지에서 잠시 제외 -> 스테이지에 git add로 재등록해야함
   -unmodified 상태 : tracked 상태이면서 스테이지에서 한 번도 수정하지 않은 원본 태

4. 파일의 상태 확인
  4.1 status 명령어로 깃 상태 확인
   - git status로 상태 확인 

5. 파일 관리 목록에서 제외 : .gitignore
  5.1 .gitignore 파일
    - 깃에서 관리하지 않는 파일들의 목록 작성 시 추적하지 않음
    - 저장소 폴더의 최상위 디렉토리에 두어야함
  5.2 .gitignore 파일 표기법
    - #으로 시작하는 줄 : 주석 처리
    - 셸 글로빙(globbing) : * 기로를 사용해 패턴 정의 ex) *.obj
    - file fullpath/fiilname : 파일 관리 제외
    - !file fullpath/fiilname : 반드시 파일 관리

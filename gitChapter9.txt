1. 리셋
 - 리셋 : 커밋을 기준으로 이전 코드로 되돌리는 방법
 1.1 복귀 시점
   - 커밋 해시 값을 이용하거나 HEAD 포인터를 이용해 상대정 위치를 지정할 수도 있음
 1.2 reset 명렁어
   - rest 명령어를 사용하면 지정된 커밋 코드로 되 돌아감
   - git reset 옵션 커밋ID 
   - [--soft] : 스테이지 영역을 포함한 상태로 복원
   - [--mixed] : 기본 옵션 값은 mixed
   - [--hard] : 실제 파일이 삭제된 이전 상태로 복원
 1.3 soft 옵션
   - 가장 낮은 단계의 리셋 동작 
   - git reset --soft HEAD~ : 이전 커밋으로 soft옵션을 사용한 리셋
   - soft 옵션은 파일을 수정하고, add 명령어로 스테이지 영역에 올려 커밋을 
     실행하기 직전의 단계로 되돌림
     => 단순히 HEAD 위치를 이동하는 역할만 
   - 소스 코드 내용이 변경되지 않더라도 새로 커밋하면 변경된 시간 같은 값을 적용해 해시 값을 생성
  1.4 mixed 옵션
   - git reset (--mixed) 커밋ID 
  1.5 hard 옵션
   - soft & mixed 옵션은 이전 커밋으로 되돌리는 과정에서 실제로 삭제되는 내용은 없음
    => 복귀 커밋 이후의 작업들은 워킹 디렉터리 영역에 남겨 두어 다시 수정하거나 커밋할 수 있도록 상태만 변경
   - hard 옵션은 리셋되는 복귀 시점의 커밋 상태와 해당 커밋의 워킹 디렉터리까지 모두 되돌림
  1.6 커밋 합치기
   * 과정
    1. git reset --soft HEAD~2 : 2단계 전의 커밋으로 리셋 
    2. 앞선 commit은 스테이지 영역에 있음
    3. git commit -m "커밋명"
  1.7 스테이지 리셋
   - git reset 파일이름 : 파일이름을 지정해 리셋하면 해당 파일은 unstage 상태로 변경
    == git reset 커밋ID 파일이름
  1.8 작업 취소
   - 수정 작업을 완전히 취소하려면 워킹 디렉토리와 스테이지 상태를 모두 제거해 마지막 커밋 상태로 되돌려 놓아야함
   - 리셋 시 시점을 현재 HEAD를 기준으로 하면 해당 시점의 수정 작업을 모두 삭제가능
  1.9 병합 취소
   - git reset --merge HEAD~ : 병합한 커밋을 취소하는 방법

2. 리버트
  2.1 취소 커밋
   - 리버트는 기존 커밋을 남겨 두고 취소에 대한 새로운 커밋을 생성
   - git revert HEAD : 현재 커밋을 리버트
  2.2 리버트 지정
   - 리버트는 한 번에 커밋 하나만 취소할 수 있음
    => 여러 커밋을 리버트하려면 최신 커밋부터 순차적으로 취소
   - git revert 커밋ID( ... 커밋ID)
  2.3 병합 취소
   - 리셋은 방금 전 실행한 병합만 삭제
   - 리버트는 시간이 지난 후에도 과거의 병합을 선택해 취소
   - git revert --mainline 숫자 병합커밋ID 
     => mainline 옵션은 병합을 취소한 후 체크아웃되는 브랜치를 표시

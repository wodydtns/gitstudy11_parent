1. remote 명령어 
 - git remote add 원격저장소별칭 URL : url 저장소에 연결하기
 - git remote rename 변경전 변경후 : 등록된 서버의 별칭 이름 변경하기
 - git remote show 원격저장소별칭 : 원격 저장소의 상세 정보 보기
 - git remote rm 원격저장소별칭 : 원격 저장소 연결 삭제

2. push 명령어
 - git push 원격저장소별칭 브랜치이름
   => 원격 저장소에 push하면 master 저장소가 총 2개 생성 / 로컬 저장소의 master 브랜치 & 서버의 master 브랜치
   => 전송은 두 브랜치 간에 전송과 수신을 동기화

3. 자동으로 내려받기 
  - git clone [url] : 원격 서버에서 정상적으로 복제 / 모든 내용을 한 번에 내려받음
  - git pull : 복제 후 원격 저장소의 갱신된 내용을 추가로 내려받으려면 사용

4. 수동으로 내려받기
  4.1 자동 병합
   - pull은 현재 커밋보다 더 최신 커밋 정보가 있을 때 내려받아 커밋 정보를 임시 영역에 저장
   - 내려받은 최신 커밋들을 현재 브랜치로 자동 병합처리
  4.2 fetch : 가져오기
   - fetch는 원격 저장소에서 코드를 수동으로 내려받음
   - fetch는 원격 저장소에서 커밋된 코드를 임시 브랜치로 내려받고 현재 브랜치와 자동 병합하지 않음
   - git fetch 원격저장소URL
 4.3 merge로 수동 병합
   - git merge 원격저장소별칭/브랜치이름

5. 순서
 5.1 최신 상태
   - 원격 저장소에 푸시하려면 자신의 로컬 저장소를 최신 상태로 유지해야함 
    => 자신의 저장소가 원격 저장소의 커밋보다 최신이어야함
   - 커밋이 순차적이지 않을 때 깃은 푸시 동작을 거부
   - 푸시 동작이 거부되면 pull or fetch 작업으로 자신의 로컬 저장소를 갱신한 후 다시 푸시해야함
 5.2 충돌 방지
   - 깃이 최신 상태에서만 푸시를 허용하는 것은 충돌 방지를 위한 것
   - 원격 저장소의 커밋을 내려받는 pull 작업은 내려받은 커밋들을 현재 브랜치로 자동 병합  
   - 권장 순서
    => [ pull -> coding 작업 -> commit -> pull -> push ]

1. 병합
 - 병합 : 분리된 브랜치를 한 브랜치로 합치는 작업
 1.1 깃을 통한 자동 병합
  - 원본을 기준으로 두 파일의 변경 이력 비교 -> 변경된 파일 내용이 발견되면 자동으로 수정된 코드 내용 병합
 1.2 병합 방식
  - 병합 시 브랜치를 만들어 브랜치 안에서 수정 작업 수행
  - 병합 방식 : Fast-Forward 병합 / 3-ways 병합

2. Fast-Forward 병합
 - Fast-Forward 방식 : 순차적 커밋에 맞추어 병합을 처리하는 방법
 2.1 병합 위치
  - git merge 브랜치이름 
  - merge 명령어는 현재 브랜치를 기준으로 다른 브랜치의 모든 커밋을 병합
  - 기준은 checkout된 현재 브랜치
   => 각 브랜치별로 파일을 다르게 관리

3. 3-way 병합
  3.1 공통 조상
   - 브랜치별로 각각 커밋하면 두 브랜치로 갈라짐
    => 이를 병합하기 위해 먼저 분할 기준인 공통 커밋을 찾고 
       콩통 조상 커밋을 포함하는 브랜치와 새로운 두 브랜치 
    => 3개를 하나로 병합 => 브랜치가 3개 있다고 해서 3-way 병합이라고 한다
   - 공통 조상 커밋을 기준으로 브랜치를 병합
   - 병합을 성공적으로 완료한 후 새로운 커밋을 추가로 하나 생성 => 병합 커밋
   - 병합 커밋은 부모 커밋이 2개
  3.2 병합 메시지
   - 두 브랜치를 병합한 후 새로운 커밋을 하면서 동시에 메시지를 자동으로 생성
   - git merge 브랜치이름 --edit : 직접 커밋 메시지를 작성할 경우 사용

4. 브랜치 삭제
  * 깃 플로 
    - 브랜치 관리 기법 / 수많은 브랜치 작업을 규격화해서 브랜치를 쉽게 다룰 수 있도록 하는 전략
    - 브랜치 이름 : master, feature,develop,release,hotfix
      => develop 브랜치는 master 브랜치에 병합한 후에도 삭제하지 않고 유지 => 이런 브랜치를 long-running 브랜치라고 함

5. 충돌
  5.1 수동으로 충돌 해결
   - 충돌 내용 수정 시 깃에서 표시한 충돌 기호도 함께 삭제
   - 충돌을 해결한 후 병합 커밋을 직접 만들어야함
  5.2 브랜치 병합 여부 확인
   - git branch --merged : 병합한 브랜치와 병합하지 않은 브랜치를 구분
   - git branch --no-merged : 병합하지 않은 브랜치를 조회

6. 리베이스
  6.1 베이스
   - 새로운 브랜치가 파생되는 커밋2 == 병합에서는 이를 공통 조상 커밋이라고 함
  6.2 베이스 변경
   - 리베이스는 코드의 베이스 분기점을 변경해 하나의 길로 만들어
     커밋의 진행 사항을 좀 더 쉽게 파악가능 하도록 함
  6.3 리베이스 vs 병합
   - 병합은 공통 조상 커밋 + 다른 두 브랜치를 순차적으로 비교해 최종 커밋 생성
   - 리베이스는 두 브랜치를 서로 비교하지 않고 순차적으로 커밋 병합 시도
     => 과정 : 공통 조상 커밋 찾기 => 베이스 커밋을 변경해 두 브랜치의 커밋 위치 변경
         => 파생된 브랜치의 diff를 임시 공간에 잠시 보관 => 기준 브랜치의 커밋을 순차적으로 진행
         => 공통 조상으로 베이스 기준점 변경 => 변경하는 기준 브랜치의 마지막 커밋에서 차례로 임시 공간에 저장한 diff를 하나씩 적용
         => 최종 브랜치까지 연장해 수정 재배치 
   * 병합과의 차이점
    - 1. 3-way 병합은 병합 커밋이 있지만, 리베이스 시 병합 커밋이 없음
      2. 브랜치의 마지막을 가리키는 커밋 위치가 다름 
   - git rebase 브랜치 : 병합 방향이 원본 브랜치에서 파생 브랜치로 향한다.
   - 리베이스는 커밋 위치를 재조정할 뿐 브랜치의 HEAD 포인터까지 옮겨 주지는 않음
     => HEAD 포인터를 일치하기 위해 Fast-Forward 병합을 실행해 주어야함
  6.4 리베이스 충돌 해결
   - 리베이스를 이용해 병합할 시 충돌된 부분들을 한 단계씩 해결해 나가면서 병합할 수 있음
  6.5 리베이스 명령어로 커밋 수정
   - [--amend] : 마지막 커밋 수정
   - [-i] : 여러 커밋을 한 커밋으로 묶을 수 있음
